# بوت شرح عقائد الإمامية

بوت تيليجرام يجيب عن الأسئلة العقائدية اعتماداً حصراً على كتاب
"دروس منهجية في شرح عقائد الإمامية" للشيخ حسين الأسدي، مع ذكر
المصدر (الجزء والصفحة) إلزامياً لكل إجابة.

## بنية المشروع

```
app/
  extract.py    — استخراج نص الكتاب من PDF وتنظيف أخطاء الترميز المكتشفة
  chunking.py   — تقطيع النص إلى مقاطع مع بيانات الاستشهاد
  embedding.py  — تمثيل دلالي محلي (fastembed، بلا تكلفة API)
  search.py     — بحث Hybrid (لفظي + دلالي)
  ai.py         — بناء الإجابة عبر DeepSeek مع استشهاد مضمون برمجياً
  database.py   — PostgreSQL (مقاطع الكتاب + المستخدمون + السجلات)
  handlers.py   — معالجات تيليجرام
  config.py     — قراءة الإعدادات من متغيرات البيئة فقط
  logger.py     — تسجيل مركزي
data/
  juz1.pdf, juz2.pdf, juz3.pdf  — ملفات الكتاب الأصلية
scripts/
  build_knowledge_base.py — يُشغَّل مرة واحدة لبناء قاعدة المعرفة
run.py — نقطة تشغيل البوت
```

## الإعداد

### 1. متغيرات البيئة

أنشئ ملف `.env` (لن يُرفع لـ Git، محمي في `.gitignore`):

```
TELEGRAM_BOT_TOKEN=توكن_من_BotFather_لـ_Shia_beliefsbot
DEEPSEEK_API_KEY=مفتاح_DeepSeek
DATABASE_URL=رابط_PostgreSQL_من_Railway
ADMIN_ID=معرّف_تيليجرام_الخاص_بك
```

**لا تضع أي قيمة افتراضية لـ ADMIN_ID في الكود** — هذا كان أحد الأخطاء
الأمنية المكتشفة في مراجعة بوت المعهد (مستودع عام + معرّف مكشوف).

### 2. تثبيت المتطلبات

```bash
pip install -r requirements.txt
```

### 3. بناء قاعدة المعرفة (مرة واحدة، أو عند استبدال ملفات الكتاب)

```bash
python3 scripts/build_knowledge_base.py
```

هذا يستخرج الكتاب من `data/*.pdf`، ينظفه، يقطّعه، يحسب المتجهات
الدلالية، ويحفظ كل شيء في PostgreSQL. يستغرق بضع دقائق عند أول تشغيل
(تحميل نموذج التمثيل الدلالي).

### 4. تشغيل البوت

```bash
python run.py
```

## قبل النشر الفعلي — نقاط يجب مراجعتها

1. **الاستشهادات بالصفحة**: رقم الصفحة المعتمد الآن = ترقيم PDF مباشرة
   (تحقق إحصائي أظهر تطابقاً في 60-90% من الحالات مع الرقم المطبوع
   الفعلي، لكن لم يُتحقق بصرياً من كل صفحة). افتح الكتاب فعلياً وقارن
   2-3 استشهادات عشوائية من إجابات البوت مع الرقم المطبوع الحقيقي
   قبل الاعتماد الكامل.
2. **عتبات القبول في `search.py`** (`DIRECT_ANSWER_SEMANTIC_THRESHOLD`,
   `DIRECT_ANSWER_LEXICAL_THRESHOLD`): قيم أولية غير مقيسة بمجموعة
   اختبار حقيقية. اجمع 20-30 سؤالاً حقيقياً بعد بدء الاستخدام واضبط
   القيم بناءً على النتائج الفعلية.
3. **جودة التمثيل الدلالي بالعربية**: لم يُختبر النموذج
   (paraphrase-multilingual-MiniLM-L12-v2) فعلياً بالعربية في بيئة
   التطوير بسبب قيود شبكة، فقط بُني الكود بشكل صحيح حسب توثيق
   المكتبة. اختبر بضعة أسئلة بلهجات مختلفة بعد أول نشر فعلي.
4. **رموز غير قابلة للاسترجاع (~400-1500 حالة لكل جزء)**: بقرار صريح
   منك، لم تُعالَج. إن ظهرت إجابة تبدو ناقصة الحرف في مكان حساس،
   راجع الصفحة الأصلية يدوياً.

## واتساب (اختياري لاحقاً)

نفس منطق `ai.py`/`search.py`/`database.py` قابل لإعادة الاستخدام؛
الفرق فقط في طبقة الاستقبال (Webhook بدل Polling). أنصح بتثبيت نسخة
تيليجرام أولاً قبل إضافة واتساب.
